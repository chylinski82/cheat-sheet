HTML & CSS cheat sheet

HTML forms tags

Tag Description Attributes
<button> Represents a clickable button, used to submit forms
or anywhere in a document for accessible, standard button functionality.		 formtarget, name, type, value
<datalist> Contains a set of option elements that represent
the permissible or recommended options available to choose from within other controls. 		only global attributes
<fieldset> It is used to group several controls as well as labels (label) within a web form. disabled, form, name
<form> Represents a document section containing interactive controls for submitting information.
<input> It is used to create interactive controls for web-based forms
in order to accept data from the user;
a wide variety of types of input data and control widgets are available,
depending on the device and user agent.
The <input> element is one of the most powerful and complex in all of HTML
due to the sheer number of combinations of input types and attributes. 		accept, alt, autocomplete, autofocuse, capture, checked,
dirname, disabled, form, formaction, formenctype, formmethod,
formnovalidate, formtarget, height, list, max, maxlength,
min, minlength, multiple, name, pattern, placeholder, readonly,
required, size, src, step, type, value, width
<label> Represents a caption for an item in a user interface. for
<legend> Represents a caption for the content of its parent fieldset. only global attributes
<meter> Represents either a scalar value within a known range or a fractional value. value, min, max, low, high, optimum, form
<optgroup> Creates a grouping of options within a select element.
<option> It is used to define an item contained in a select, an optgroup, or a datalist element.
As such, <option> can represent menu items in popups and other lists of items
in an HTML document. 		disabled, label
<output> It is a container element into which a site or app can inject the results
of a calculation or the outcome of a user action. 		for, form ,name
<progress> displays an indicator showing the completion progress of a task,
typically displayed as a progress bar. 		max, value
<select> Represents a control that provides a menu of options. autocomplete, autofocus, disabled, form, multiple,
name, required, size
<textarea> Represents a multi-line plain-text editing control,
useful when you want to allow users to enter a sizeable amount of free-form text,
for example a comment on a review or feedback form. 		autocapitilize, autocomplete, autocorrect, autofocus, cols,
disabled, form, maxlength, minlength, name, placeholder,
readonly, required, rows, spellcheck, wrap

CSS flex properties

Property Description Values
align-content Sets the distribution of space between and around content items
along a flexbox's cross-axis or a grid's block axis. 		center, start, end, flex-start, flex-end, normal, baseline, first baseline,
last baseline, space-between, space-around, space-evenly, stretch,
safe center, unsafe center
align-items Sets the align-self value on all direct children as a group.
In Flexbox, it controls the alignment of items on the Cross Axis.
In Grid Layout, it controls the alignment of items on the Block Axis within their grid area. 		normal, stretch, center, start, end, flex-start, flex-end, baseline,
first baseline, last baseline, safe center, unsafe center
align-self Overrides a grid or flex item's align-items value.
In Grid, it aligns the item inside the grid area.
In Flexbox, it aligns the item on the cross axis. 		auto, normal, center, start, end, self-start, self-end, flex-start,
flex-end, baseline, first baseline, last baseline, stretch,
safe center, unsafe center
flex CSS shorthand property sets how a flex item will grow or shrink
to fit the space available in its flex container. 		Constituents properties (flex-grow, flex-shrink, flex-basis)
flex-grow Sets the flex grow factor of a flex item's main size The flex-grow property is specified as a single <number>
flex-shrink Sets the flex shrink factor of a flex item.
If the size of all flex items is larger than the flex container,
items shrink to fit according to flex-shrink.
In use, flex-shrink is used alongside the other flex properties flex-grow and flex-basis,
and normally defined using the flex shorthand. 		The flex-shrink property is specified as a single <number>
flex-basis Sets the initial main size of a flex item.
It sets the size of the content box unless otherwise set with box-sizing. 		<width in em, px>, auto, fill, max-content, min-content, fit-content,
content
flex-direction Sets how flex items are placed in the flex container
defining the main axis and the direction (normal or reversed). 		row, row-reverse, column, column-reverse
flex-flow Specifies the direction of a flex container, as well as its wrapping behavior. row, row-reverse, column, column-reverse, nowrap, wrap, wrap-reverse,
row nowrap, column wrap, column-reverse wrap-reverse
flex-wrap Sets whether flex items are forced onto one line or can wrap onto multiple lines.
If wrapping is allowed, it sets the direction that lines are stacked		 nowrap, wrap, wrap-reverse
justify-content Defines how the browser distributes space between and around content items
along the main-axis of a flex container, and the inline axis of a grid container. 		center, start, end, flex-start, flex-end, left, right, normal, space-between,
space-around, space-evenly, stretch, safe center, unsafe center
order Sets the order to lay out an item in a flex or grid container.
Items in a container are sorted by ascending order value and then by their source code order 		<integer> values
